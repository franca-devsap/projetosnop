# /fiori-frontend-build

1. Read root context first, then the nearest customer `CLAUDE.md`, then the nearest app `CLAUDE.md` if one exists.
2. Read the governing design source: customer workbook first, then app docs, then root `docs/DESIGN_SYSTEM.md`.
3. Confirm whether the task is visual, structural UI, or both.
4. If the app is SAPUI5 freestyle, pair this skill with `/fiori-ui5-freestyle-build`.
5. Reuse approved customer patterns before inventing new ones.
6. Verify responsive behavior, focus states, and accessibility.
7. Finish with `/fiori-frontend-review`.
